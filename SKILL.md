# 课程考试资料整理（course-exam-prep）

帮用户把一门课的课件整理成期末能用的资料。课件多、上下文长，所以靠**一个自包含
的规则文件**统一口径，再用 **subagent（每讲独立 context）**把各讲分开整理，避免
所有讲的内容堆进同一个上下文里爆掉。

## 整体心智模型：建规则+试点 → 批量派发 → 编译

```
阶段0 建规则 + 试点第一讲（一个对话内一气呵成）
   问信息 → 读「试点讲」课件 → 分析题型/知识单元 → 生成《规则》/汇总骨架/manifest
   → 紧接着把这一讲整理出来（试点）→ 和用户过一遍、调到满意（踩的坑写进规则文件）
        │  规则与模板已被这一讲验证过
阶段1 派 subagent 整理剩余各讲（规则验证后才做）
   每讲派一个独立 subagent（独立 context，防主线程爆炸）→ 各只产自己的 第NN讲_*.md
   → 串行合并：单一写入者把各讲并进 00_ 汇总 / 勾进度 / 记坑
        │
阶段2 编译成品（一次）
   build_word.py 合并所有 md → 可打印的 Word 手册
```

为什么先试点再派发：建规则本就要读一讲课件，顺手把这讲整理了，能在"把同一个坑
放大 N 倍"之前就发现并写进规则文件——之后并行的 subagent 冷启动直接读到、不重蹈。
**派 subagent 必须用户确认试点结果 OK 后才做**（也只在用户认可批量整理时才派）。
规则文件必须自包含：subagent / 日后续整理只读它就能冷启动干活。

## 模式

- **开卷**：做"翻得快"的纸质速查手册。求全、宁多勿漏、排序索引。详见
  `references/open-book.md`，规则模板 `assets/rules_template.md`。
- **闭卷**：做"记得住 + 想得通 + 能应用"的复习材料。覆盖**记忆 / 理解 / 应用**
  三层——尤其算法/理工课大头在理解和应用（推导、解释、选方法），**别做成纯
  背诵卡**。产物含掌握分级、概念/推导卡、应用迁移题、自测题。**本模式不做总表、
  不做闪卡**（掌握分级靠每讲 ★掌握清单、公式靠每讲"公式 & 推导"节）。详见
  `references/closed-book.md`，规则模板 `assets/rules_template_closed.md`。

建规则时用 AskUserQuestion 问清本课程是开卷还是闭卷，选对应模板写进规则文件。
不要默认——开/闭卷的取舍和产物完全不同。

## 阶段0：建规则 + 试点第一讲（一个对话内做完）

目标产物（都放在用户的课程目录，如 `D:/某课/`）：
1. `整理笔记/00_整理规则.md` —— 后续每讲整理的唯一依据，必须自包含（subagent 只读
   它就能干活）。开卷用 `assets/rules_template.md`、闭卷用 `assets/rules_template_closed.md`
   作模板，把课程参数填实。
2. 汇总文件骨架：
   - 开卷：`00_总索引.md`，以及（按需）`00_思考题汇总.md`、题型专项总表。
   - 闭卷：`00_自测题库.md`（**本模式不做总表/闪卡**）。
3. `_manifest.json` —— 给 build_word.py 用，定标题/顺序/排序（闭卷加 `mode:closed`
   和文件 role）。开卷参考 `assets/manifest_example.json`，闭卷参考
   `assets/manifest_example_closed.json`。

做法：
1. **问清基本信息**：课程名、考试形式（开/闭卷）、课件在哪、共几讲、什么格式，
   并选一讲作**试点**（通常就是建规则时要读的那一讲，第1讲或内容典型的一讲）。
   **不确定的偏好一律用 AskUserQuestion 问用户，不要自己揣测。** 至少要明确问：
   - **笔记语言**：全中文 / 中文为主+术语附外文 / 中英对照。这是 per-course 偏好，
     上一门课怎么设跟这门课无关，必须当面问，别照搬别的课或记忆里的设定。
   - 其它拿不准的（如要不要附年代、是否收录思考题大题）也用提问确认。
2. **分析小测/样题**：找出题型和侧重点。
   - 开卷：决定每讲建哪几张速查表、要不要题型专项区（如听辨题→歌单）。
   - 闭卷：判断**记忆/理解/应用三层的权重**——算法理工课大头常在理解与应用
     （推导、解释、选方法、计算），别把材料做成纯背诵卡。
   让用户给一两份小测或描述题型。
3. **确定本课"知识单元/实体类别"**：
   - 开卷看最常考的实体（人物/作品？事件/概念？法条/案例？），命名速查表与索引分表。
   - 闭卷看知识单元（概念/公式推导/算法机理/对比），定每讲哪几节是重头。
4. 用对应模式的模板生成规则文件、汇总骨架、manifest。
5. **紧接着把试点这一讲整理出来**（按下面「单讲整理细则」，自己做、不派 subagent）。
   课件文字此时已读，直接提取补全→写 `第NN讲_*.md`→并进 `00_` 汇总→勾进度。
6. **和用户过一遍试点成果**：模板节够不够用、详略、语言、公式/图处理对不对。
   据反馈改规则文件和这讲笔记，直到用户满意。**这一步把这门课的坑都暴露出来，
   写进规则文件「⚠ 踩坑记录」区**——后面派的 subagent 冷启动就能直接避开。

完成后告诉用户："规则建好了、第N讲作为试点也整理好了。你确认没问题后，我可以派
subagent 把剩下的讲都整理掉（每讲独立、互不干扰）。"

## 阶段1：派 subagent 整理剩余各讲（试点确认 OK 后）

**前提**：阶段0 的规则 + 试点讲已让用户确认满意，坑已写进规则文件。**只有用户认可
批量整理时才派 subagent。** 核心：每讲一个独立 subagent（独立 context，主线程不爆），
各只产自己的文件；所有共享文件留到最后由单一写入者合并——零竞态。

### 为什么这样设计
- subagent 有独立上下文：提取课件、看图、推理全留在它自己 context 里，**只把一条
  短总结返回主线程**。哪怕整门课，主线程也始终轻。
- 文件系统共享、**无跨 agent 锁**。多 agent 同时往同一个 `00_` 汇总或规则文件写
  = 读-改-写竞态，互相覆盖丢更新。所以**按文件分工**：
  | 文件 | 谁写 | 并行安全 |
  |---|---|---|
  | `第NN讲_笔记.md`（闭卷不做闪卡） | 各讲自己的 subagent | ✅ |
  | `00_*` 汇总、规则文件（进度/踩坑） | 仅合并步骤，单一写入者 | ✅ |
- ⚠️ 不要用 `isolation:"worktree"`——它靠 git 合并，课程目录通常不是 git 仓库。
  正解是"分文件 + 串行合并"，共享同一目录即可。

### 步骤
1. **确定还剩哪些讲（只派没做过的）**：**以课程目录里已存在的 `第NN讲_*.md` 为准**
   （比进度表勾选更可靠——用户可能整理了却没勾）。把"全部讲"减去"已有笔记文件的讲"
   就是待整理集；规则文件进度表作为参考交叉核对。常见情形：用户已先整理了若干讲
   （如试点讲、或早先自己做的第2/3讲），**这些一律跳过**，只给缺的讲派 subagent，
   绝不重复劳动、绝不覆盖已整理内容。拿不准某讲算不算"做好了"就问用户。
2. **并发派发**：一条消息里为每讲派一个 Agent（量大分批，一批 3–5 个，避免速率/
   成本问题）。派发指令用 `assets/subagent_prompt_template.md` 填实（课程目录、讲号、
   课件路径、规则文件路径）。**硬约束**写进指令：
   - 自包含：自己读规则文件 + 对应模式 reference（subagent 不会自动加载本 skill）。
   - 只写**自己那讲**的 `第NN讲_笔记.md`（闭卷也只产这一个、**不做闪卡**）。
   - **绝不编辑任何 `00_*` 文件、也不改规则文件**。
   - 只回一条短总结：写了哪些文件 + 本讲踩的坑（供合并记录）。
3. **串行合并（所有 subagent 跑完后，单一写入者一次做完）**：小课程主线程自己做；
   讲数很多想省主线程 context 时可只派**一个**合并 subagent 串行做。
   - **从所有 `第NN讲_*.md` 整体重建 `00_` 汇总**（幂等：结果是全部已整理讲的并集，
     不论某讲是试点时做的还是 subagent 做的，重建都不会重复）：
     - 开卷：`00_总索引.md`（同名只留一行）/ `00_思考题汇总.md` / 专项总表。
     - 闭卷：自测题→`00_自测题库.md`（**本模式不做总表/闪卡**）。
   - 在规则文件进度表里把这批讲一次性勾上。
   - 把各 subagent 报回的坑统一写进规则文件「⚠ 踩坑记录」区。
4. 全部整理完 → 进阶段2 编译。

## 单讲整理细则（阶段0 试点 / 每个 subagent 都照此）

整理某一讲时：
1. **读规则文件** `整理笔记/00_整理规则.md`，按它走（含「⚠ 踩坑记录」，照着避坑）。
2. **提取课件**——两条腿走路，别只抽文字：
   - 先抽文字：`PYTHONIOENCODING=utf-8 python <skill>/scripts/extract.py "课件/第N讲.pdf"`
   - 看输出末尾的「文字量统计」，对⚠️低文字页（内容多在图里）渲染成 PNG 再**用
     Read 工具看图**补全：`extract.py "..." --images "img/第N讲" --lowtext-only`
   - 详见 `references/extraction.md`。**这一步是正确性关键**：乐谱、示意图、
     扫描页、图表文字只能靠看图拿到，漏了会出错。
3. **写笔记** `整理笔记/第NN讲_标题.md`，按规则文件指定模式的模板：
   - 开卷 → `references/open-book.md`：带★的速查表要全、写"易混淆/不包含"列。
   - 闭卷 → `references/closed-book.md`：覆盖记忆/理解/应用三层，写"为什么/直觉"、
     推导主线、应用迁移题。**别做成纯背诵卡。**（**本模式不做闪卡**，每讲只产一个笔记文件。）
   - 两者都要：Word 兼容写法（只用 #/##/###、- 列表、表格、**加粗**）。
4. **共享文件怎么处理——分两种身份**：
   - **阶段0 试点（主线程亲自整理这一讲）**：此刻你就是单一写入者，直接把本讲并进
     `00_` 汇总（开卷→`00_总索引`等；闭卷→`00_自测题库`，本模式不做总表/闪卡）、
     在规则文件进度表勾上本讲、把踩的坑写进「⚠ 踩坑记录」。
   - **subagent（阶段1 批量）**：**绝不碰任何 `00_` 文件和规则文件**，只产自己那讲的
     `第NN讲_*.md`，把踩的坑**报回**主线程；共享文件由阶段1 的合并步骤统一写。

## 阶段2：编译成可打印手册（全部整理完后一次）

```bash
PYTHONIOENCODING=utf-8 python <skill>/scripts/build_word.py "<课程笔记目录>"
```
脚本读目录下 `_manifest.json`：
- 开卷（默认）：产出便于纸质速查的 Word——封面+三步速查法、**总索引按拼音排序**
  （各讲正文表保持课程顺序）、每讲另起一页、页眉显示讲名、表头跨页重复、页脚页码。
  自检与坑见 `references/open-book.md`。
- 闭卷（manifest 里 `mode:closed`）：产出复习手册——自测题另起页加"先做后看"提示、
  按讲顺序不排序（本模式不做总表/闪卡）。自检与坑见 `references/closed-book.md`。

## 关键原则（每个阶段都适用，做错过的坑）

- **编码**：所有 python 调用前加 `PYTHONIOENCODING=utf-8`，否则 Windows GBK
  下中文会 `UnicodeEncodeError`。
- **图片里的内容会被漏**：纯文字提取拿不到图中信息，务必对低文字/图示页看图。
- **PPT 公式抓不到**：`extract.py --images` 对 pptx 只导内嵌图、**抓不到公式对象**。
  有公式/矩阵/示意图的 PPT 先用 `scripts/pptx_to_pdf.py` 转 PDF，再对 PDF
  `--images` 整页渲染看图。详见 `references/extraction.md`。**公式密集的 slide 课，
  PPT→PDF 要在主线程一次性批量转好再派 subagent**——PowerPoint COM 是单实例，多
  subagent 并行转会互相卡死；subagent 只用现成 PDF 渲染。
- **表格内公式禁用字面 `|`**：会按列裂表；绝对值用 `\lvert x \rvert`、范数用
  `\lVert x \rVert`。表格还必须带 `|---|` 分隔行，否则不渲染成表格。
- **理工/数学课公式要在 Word 里渲染**：`build_word.py` 已会把笔记里的 `$...$` LaTeX 转成
  Word **原生公式（OMML）**（latex2mathml→MathML→Office MML2OMML.XSL→OMML）。**没装
  `latex2mathml` 会回退成字面 `$x$` 文本、公式没法看**，有公式的课编译前务必装。验证：
  解压成品 docx 数 `word/document.xml` 里 `<m:oMath>` 个数、确认无残留含 `$...$` 的 `<w:t>`。
  公式定界符**只用 `$...$` / `$$...$$`，禁用 `\(...\)` / `\[...\]`**（派 subagent 时强调）——
  写成 `\(...\)` 会整讲公式不渲染、变字面文字；脚本虽已加兜底归一，但源头统一 `$...$` 才稳。
- **踩坑要回写规则文件**：每讲解决的问题写进规则文件「⚠ 踩坑记录」区，让后续
  冷启动对话直接受益、别重复踩坑。
- **音频/视频不抽取、不试听**：听辨题答案课件文字里通常已写明，照文字整理。
- **跨对话自包含**：规则文件要让冷启动对话只读它就能干活；别依赖上一个对话的记忆。
- **per-course 偏好别硬套**：笔记语言（中/中英）、实体类别、题型都按当前课程定，
  不要把上一门课的设定照搬过来。
- **Word 兼容**：笔记只用 #/##/###、- 列表、表格、**加粗**；不用 HTML/脚注/嵌套表。
- 依赖一次装好：`pip install pymupdf python-pptx python-docx pypinyin latex2mathml`
  （`latex2mathml` 用于把 `$...$` 公式渲染成 Word 原生公式，理工/数学课必装）。
