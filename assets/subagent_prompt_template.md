# 单讲 subagent 派发指令模板（阶段1 批量整理用）

阶段0 的规则 + 试点讲确认 OK 后，主线程为**每一讲剩余的讲**派一个 subagent，用下面
模板填实后作为该 agent 的任务（`prompt`）。一条消息里可并发派多个（量大分批，一批
3–5 个）。

关键：subagent 有独立 context、不会自动加载本 skill，所以指令必须**自包含**——
明确叫它读规则文件和对应 reference，并**硬约束它只写自己那讲的文件、绝不碰共享
文件**（避免多 agent 同写竞态）。

---

## 填空模板（把 `<…>` 换成实际值再派发）

```
你在帮我整理《<课程名>》第 <N> 讲的期末复习笔记。这是一个隔离任务，请严格按下面做。

【先读这两个文件，按它们走】
1. 规则文件（唯一依据，自包含）：<课程目录>/整理笔记/00_整理规则.md
2. 笔记模板与规范：<skill 路径>/references/<闭卷填 closed-book.md / 开卷填 open-book.md>
   开头也读一下 <skill 路径>/references/extraction.md 的提取方法。

【本讲课件】
<课程目录>/<课件相对路径，如 课件/第N讲.pptx>

【提取（两条腿：文字 + 看图）】
- 先抽文字：PYTHONIOENCODING=utf-8 python <skill 路径>/scripts/extract.py "<课件路径>"
- 看末尾「文字量统计」，对⚠️低文字页/图示页渲染 PNG 再用 Read 工具看图补全。
- ⚠️ 若是 PPT 且含公式/矩阵/几何示意：extract.py 抓不到公式对象，改用
  python <skill 路径>/scripts/pptx_to_pdf.py "<课件路径>" 转成 PDF，
  再对该 PDF --images 整页渲染、Read 看图。
- 规则文件「⚠ 踩坑记录」区里若已记了本课的坑，照着避。

【写笔记——只写下面这些"属于本讲"的文件，按模板规范】
- <课程目录>/整理笔记/第<NN>讲_<标题>.md
  （闭卷模式只产这一个文件，**不做闪卡**；开卷同理只产本讲速查文件）
- Word 兼容：只用 #/##/###、- 列表、表格、**加粗**；表格必须带 |---| 分隔行；
  表格单元格里的公式不能含字面 |（绝对值用 \lvert x \rvert、范数用 \lVert x \rVert）。
- ⚠️ 公式定界符**只用 `$...$`（行内）/ `$$...$$`（行间）**，**禁用 `\(...\)` 和 `\[...\]`**。
  编译脚本默认只认 `$...$`，写成 `\(...\)` 会导致整讲公式在 Word 里不渲染、变成字面文字。

【硬约束（务必遵守，避免和其它并行 agent 冲突）】
- 绝不创建或编辑任何 00_ 开头的汇总文件（开卷如 00_总索引；闭卷如 00_自测题库 等）。
- 绝不编辑规则文件 00_整理规则.md（包括进度勾、踩坑记录）。
  这些共享文件由主线程在合并阶段统一写，你只管产出自己这讲的文件。

【完成后只回一条简短总结，不要贴长正文】
- 写了哪些文件（文件名）。
- 本讲整理时踩的坑 / 做的特殊处理（如某页公式靠 PDF 渲染才拿到、某图内容是…）
  ——一两行即可，供主线程写进规则文件的踩坑记录。
- 本讲产出里要并入 00_ 汇总的关键条目数量（如 ★★★ 几条、自测题几道），方便核对。
```

## 派发前（主线程）：只派没做过的讲
派之前先看课程目录里已存在哪些 `第NN讲_*.md`（以磁盘文件为准，比进度表勾选更可靠），
**已有的讲就别派了**，只给缺的讲派 subagent，避免重复劳动 / 覆盖已整理内容。

## 派发后（主线程）
所有讲 agent 跑完 → 进 SKILL.md 阶段1 的合并步骤：单一写入者一次性**从所有
`第NN讲_*.md` 整体重建** `00_*` 汇总（幂等、不重复）、勾进度、把各 agent 报回的坑
写进规则文件踩坑记录。
