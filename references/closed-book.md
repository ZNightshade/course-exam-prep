# 闭卷模式：做"记得住 + 想得通 + 能应用"的复习材料

闭卷考试没有材料在手，但它**绝不只考默写**。尤其算法 / 理工 / ML 这类课，
大头是**理解和应用**：能不能推导、能不能解释一个现象、能不能在没见过的场景
里选对方法并说清为什么。所以闭卷材料要同时支撑三件事：

- **记得住**——核心定义、公式、步骤能调取出来（默写型题）。
- **想得通**——知道每个方法**为什么这么做、机理是什么**，能复现推导思路、
  能用自己的话解释（理解型题：解释/证明/"为什么"）。
- **能应用**——给一个新情形能选对方法、判断后果、迁移套用（应用型题：
  案例分析/计算/"如果……会怎样"）。

> ⚠️ 最大的坑：把闭卷做成纯背诵卡。**理解和应用不能丢**——它们往往是
> 大题、是拉分项。背得出公式但不懂推导、记得住步骤但不会选用，照样丢分。

> ⚠️ **本模式不做"总表"和"闪卡"**（per-course 默认）：
> - **不做** `00_掌握总表.md` / `00_公式推导总表.md` 这类跨讲汇总总表——掌握分级靠
>   每讲开头的 ★掌握清单，公式靠每讲"公式 & 推导"一节即可。
> - **不做闪卡**（不产 `第NN讲_闪卡.md`、不产 `00_闪卡总集.md`）。主动回忆靠每讲
>   第八节「自测题」+ 全课程 `00_自测题库.md`。
> - **保留自测题**：每讲自测题 + 汇总自测题库（含参考习题/真题解析）。
> 若某门课确实想要总表或闪卡，建规则时再单独和用户确认、按需加回，别默认。

> 闭卷和开卷共用前 70~80% 的流程（提取课件、分析小测题型、按讲结构化整理、
> 汇总总知识库、跨对话工作流），脚本 `extract.py` 完全复用。差异集中在
> **取舍**（求精分主次，不是求全）和**最终产物**（复习材料，不是速查表）。

| | 开卷 | 闭卷 |
|---|---|---|
| 目标 | 快速查到 | 记住 + 理解 + 会应用 |
| 取舍 | 求全，宁多勿漏 | 求精，分主次（标必掌握/重点/了解）|
| 产物 | 排序索引 + 速查手册 | 每讲掌握分级 + 概念/推导卡 + 应用题 + 口诀 + 自测题 + 自测题库 |
| 特色 | "易混淆/不包含"、拼音索引 | "为什么/直觉"、能复现推导、场景迁移、主动回忆 |

## 一、每讲笔记模板（按本课程实际调整）

每讲一个 `第NN讲_标题.md`。骨架覆盖**记忆 / 理解 / 应用**三层——别只填记忆层。
具体哪几节是重头，看本课程"知识单元"和小测题型（见"按课程定制"）。

```markdown
# 第NN讲：标题
> 一句话主旨 + 本讲在课程里的位置（承上启下）

## 🎯 ★掌握清单（先掌握什么、要掌握到什么程度）
| 知识点 | 优先级 | 掌握程度 | 一句话抓手 |
| 核心定义X | ★★★ | 记忆+理解 | … |
| 算法Y | ★★★ | 理解+会应用 | … |
| 推导Z | ★★ | 能复现思路 | … |
（优先级依据小测/老师强调；"掌握程度"明确是"会背/要懂/能用"——闭卷不是
 所有东西都靠背，理解型的标"理解"，应用型的标"会应用"）

## 一、本讲框架（脑图式主干）
（3–5 条主干 + 它们之间的逻辑关系，先建立结构理解，再填细节）

## 二、★核心概念卡（是什么 / 为什么 / 怎么用）
（每个概念三件套，重点在"为什么"——闭卷理解题就考这个）
- **概念A**
  - 是什么：一句定义（默写用）
  - **为什么/直觉**：动机、它解决什么问题、背后的道理（要能用自己话讲）
  - 怎么用 / 边界：什么时候用、什么时候失效

## 三、★公式 & 推导（不是背结果，是会复现）
| 公式 | 符号含义 | 推导主线（关键几步+为什么） | 易错点 |
（闭卷要能**重推一遍**而非死记结论：写清推导是怎么一步步来的、每步的依据；
 这样即使忘了结果也能现场推出来，还能答"证明/推导"大题）

## 四、★方法 / 算法（步骤 + 为什么 + 何时用）
| 方法 | 步骤(编号) | 每步在做什么/为什么 | 适用条件 | 优缺点 |
（不只背步骤顺序，要懂每步的作用——考"解释算法""改一步会怎样"时才答得出）

## 五、★对比辨析表（简答最爱考"区别 / 选哪个"）
| 对比维度 | A | B | 何时选 A / 选 B |
（易混概念两两对比；最后一列"何时选"把辨析升级成应用判断）

## 六、★应用 / 迁移（给情形→选方法+讲理由）
（本讲方法的典型应用场景、例题、"如果数据变成…会怎样"。这一节专练
 "把知识用出去"，是闭卷大题/计算题的命门，别省）
- 场景：…… → 该用什么 + 为什么 + 怎么算

## 七、记忆口诀 / 联想（给纯记忆项用）
（只给"确实得硬记"的并列项/易混项编口诀；能理解推出的就别硬背）

## 八、★自测题（回忆 / 理解 / 应用 三类都要）
（5–10 题，覆盖三层：① 回忆型"X是什么/默写公式" ② 理解型"为什么/解释/
 证明" ③ 应用型"给定情形选方法/计算/如果…会怎样"。答案集中放节末"答案"
 小标题下，先做后看）

## 九、本讲一句话总结（机理层面，不是名词堆砌）
```

> 本模式**不产闪卡文件**，每讲只产 `第NN讲_标题.md` 一个文件。

### 为什么"掌握分级 + 三层"是闭卷的命门
闭卷时间有限，要把劲用在刀刃上。每个知识点既标**优先级**（★★★/★★/★），
也标**掌握程度**（会背 / 要懂 / 能用）——这才说清"这个只要记住、那个必须吃透、
另一个得会套用"。只标"必背"会误导你把理解型考点也当默写题刷，考场上照样卡壳。

## 二、按课程定制（建规则阶段要做的判断）
不同课程的三层权重不同，看小测题型定侧重：
- **理工/算法课（如数据科学、机器学习）**→ **理解 + 应用为主**：第三节推导、
  第四节算法机理、第六节应用迁移是重头；纯记忆口诀节最轻。考的是"推导/解释/
  选方法/计算"，把材料做成"会推会用"，不是"背得出"。
- **概念/记忆课（历史/政治/部分文科）**→ 记忆层加重：概念卡、时间线、口诀为主，
  但"为什么/影响"的理解题仍要留。
- 判断依据：**看小测/样题考什么**——考默写就强化公式记忆；考"证明/解释/为什么"
  就强化推导主线和概念卡的"为什么"；考"比较/选用/计算"就强化对比表和应用节。

## 三、记忆口诀 vs 理解（别滥用口诀）
口诀只对"**确实得硬记、又没逻辑可推**"的东西有用（如约定俗成的命名、并列清单）。
能靠理解推出来的（公式、步骤的道理）就别编口诀硬背——理解记得更牢、还能应对变体。
- 并列项 → 取首字编顺口句；易混对 → 一句话点关键差异 + 画面联想。
- 口诀写进每讲第七节。

## 四、前置汇总文件（全课程共用，放手册最前面）
本模式**只做一个汇总**：
- `00_自测题库.md`：各讲自测题汇总（回忆/理解/应用三类），考前模拟；答案放末尾。
  另设"参考习题"专区，把老师给的习题 / 往年真题 + 答案整理成带解析的题目，对折遮挡答案。

> 编译时这个 `00_` 文件默认不排序，按讲顺序更利于顺着结构理解记忆。
> （本模式不做 `00_掌握总表.md` / `00_公式推导总表.md` / `00_闪卡总集.md`。）

## 五、Word 兼容写法（成品要进 Word，别用花哨语法）
只用：`#`/`##`/`###` 标题、`-` 列表、Markdown 表格、`**加粗**`。
不要用：HTML 标签、脚注、嵌套表格、图片外链。`<details>` 这类 build_word 不解析，
自测题答案请用"答案"小标题集中放节末，而不是 HTML 折叠。
**⚠️ 表格必须带 `|---|---|` 分隔行**（本文档示例为简洁常省略它，真正写进笔记
时一定要加），否则 build_word 不会把它渲染成表格，会当成普通段落。
**⚠️ 表格单元格里的 LaTeX 公式绝不能含字面 `|`**（表格按 `|` 分列，会裂表）：
绝对值 `|x|` 写成 `\lvert x \rvert`，范数 `\|x\|` 写成 `\lVert x \rVert`，
渲染效果相同但不含 `|` 字符。公式节里写公式时直接这样写，别事后修。

## 六、编译成复习手册
全部讲整理完后，用脚本合并（与开卷同一个脚本，靠 manifest 区分模式）：
```bash
PYTHONIOENCODING=utf-8 python <skill>/scripts/build_word.py "<课程目录>"
```
`_manifest.json` 里把 `"mode": "closed"`，并给文件标 `role`：
- `role: "selftest"` —— 自测题库，另起页加"先做再看答案"提示。
- `role: "unit"` —— 每讲正文，不排序（保结构）。

（本模式不用 `role: "memorize"`（总表）和 `role: "flashcard"`（闪卡）。脚本仍保留这两种
 渲染能力，只是闭卷默认不产生这两类文件、manifest 里也不列。）

`_manifest.json` 示例见 `assets/manifest_example_closed.json`。

**公式渲染（理工/数学课必看）**：`build_word.py` 会把笔记里的 `$...$` / `$$...$$` LaTeX
转成 Word **原生公式（OMML）**，所以编译前务必 `pip install latex2mathml`（链路用到它 +
Office 自带的 MML2OMML.XSL）。**没装会回退成字面 `$x$` 文本**，整本手册公式都是源码、没法看。
验证：解压成品 docx，数 `word/document.xml` 里 `<m:oMath>` 个数、并确认没有残留含 `$...$`
的 `<w:t>`（有残留说明那条公式转换失败、回退了）。
> ⚠ **公式定界符只用 `$...$` / `$$...$$`，禁用 LaTeX 的 `\(...\)` / `\[...\]`**。
> 派 subagent 写笔记时务必在指令里强调这条——曾有一讲整篇写成 `\(...\)`，而 build_word.py
> 默认只切 `$...$`，结果该讲公式全被当普通文字原样落进 Word、大面积没渲染。
> （脚本现已加兼容层把 `\(...\)`/`\[...\]` 归一成 `$...$` 兜底，但**源头统一写 `$...$` 才是正道**，
> 也便于 KaTeX 预览。合并某讲到自测题库前，先 `grep '\('` 抽查有没有漏网的。）
> ⚠ **范数/绝对值方向定界符 bug（脚本已自动修，知道即可）**：`\lVert/\rVert/\lvert/\rvert`
> 会被 Office 的 MML2OMML.XSL 当成对的伸缩定界符 `<m:d>`，**紧跟下标/上标时内容被吞成空**
> （`\lVert x\rVert_2` 只剩 "2"）。`latex_to_omml` 已在转换前把它们换成无方向的 `\Vert/\vert`
> 解决。验证捷径：解压成品数 `word/document.xml` 里 `<m:e/>`（空数学元素）应为 **0**。
> 闭卷偏好按课定（per-course）：默认不做总表/闪卡；题目是否做**中英双语题干**（老师出
> 英文题时很有用，英文在前练读题、答案留原语言）也问用户，别硬套上一门课。

### 成品自检
- 每讲是否三层都覆盖（记忆 / 理解 / 应用），不是清一色默写？
- 概念卡的"为什么/直觉"是否写到能讲明白机理，而不只是定义？
- 公式节是否给了**推导主线**（能重推），而不只是结论？
- 算法节是否讲了每步"为什么"和"何时用"，而不只是步骤顺序？
- 是否有应用/迁移题（场景→选方法+理由）？自测题三类齐不齐？
- 每讲 ★掌握清单的"掌握程度"列是否区分了"会背/要懂/能用"？
- 确认**没有产出总表/闪卡文件**（只有每讲正文 + `00_自测题库.md`）。

### 已知坑
- **把闭卷做成纯背诵卡**——最常见、最致命。理解题和应用题才是大题失分重灾区，
  务必保留"为什么 / 推导主线 / 场景选用"。
- 公式只抄结论不写推导——一旦忘了结论就全盘崩，且答不了"证明/推导"题。
- 自测题答案和题目同屏——起不到主动回忆作用；用"答案"小标题集中放节末。
- 滥用口诀去硬背本可理解的内容——记不牢也应付不了变体。
- `<details>`/HTML 折叠在 Word 里不生效，用底纹或分节代替。
- 其余 Word 域/字体/锁文件等坑与开卷一致，见 `open-book.md`「已知坑」。
