# 开卷模式：做"翻得快"的纸质速查手册

开卷考试有材料在手，比的是**多快能翻到答案**。纸质版没有 Ctrl+F，所以一切
设计都围绕"像查字典一样定位"。核心三件事：**前置排序索引**、**每讲结构固定**、
**成品分页+页眉+页码**。

## 一、每讲笔记模板（按本课程实际调整）

每讲一个 `第NN讲_标题.md`。下面是通用骨架——**带★的几张速查表是开卷的命门**，
具体分几张表、列什么列，要根据本课程最常考的"实体类别"来定（见"按课程定制"）。

```markdown
# 第NN讲：标题
> 时期/范围 + 一句话主旨

## 🔑 速查关键词
（本讲所有关键名词堆在一行，方便整页扫读）

## 一、本讲框架
（教学内容/目标，2–4 条）

## 二、核心知识点（按课件顺序）
（分点罗列，保留课件顺序，便于和原课件对照）

## 三、名词解释 / 术语表
| 术语 | 解释（1–2句，开卷能直接抄） | 备注/关联 |

## 四、★【实体类别A】速查表
（例：人物表 / 作曲家表。列：名称 | 时期/国别 | 标签/称号 | 代表成果）

## 五、★【实体类别B】特征速查表
（例：体裁/流派/概念特征表。列：名称 | 时期 | 关键特征 | 易混淆/不包含）

## 六、★【实体类别C】速查表
（例：作品/案例/事件表。列：名称 | 归属 | 类别 | 关键细节）

## 七、易考点 & 小测关联
（用 📌 标出"最…/首创/年份/特例/常考多选"，对齐小测题型）

## 八、（题型专项区，按需）
（特殊题型的素材，如听辨歌单/必背公式/识图要点，见"题型专项"）

## 九、课后思考题（含参考答案）⭐
（若老师从思考题出大题，把每题完整参考答案写全，能直接照抄）

## 十、本讲一句话记忆点
```

### "易混淆/不包含"列为什么重要
开卷选择题最爱考"以下哪项**不**属于X"。把每个体裁/概念的"不包含什么、和谁
易混"直接写在表里，考场上对着读一眼就能排除。这一列是开卷拿分的关键。

## 二、按课程定制（建规则阶段要做的判断）
不同课程的"实体类别"不同，第四到六节要据此命名：
- 音乐史 → 作曲家 / 体裁流派 / 作品
- 通史/政治史 → 人物 / 事件 / 概念制度
- 法学 → 法条 / 学说 / 案例
- 文学 → 作家 / 流派 / 作品
判断依据：**看小测/样题考什么实体**，就为那几类各建一张速查表。

## 三、题型专项区（看小测决定要不要）
小测里若有"特殊题型"，单建一节/一张总表收集素材：
- **听辨/音响题**（音乐课）→ 必听歌单表：曲目 | 作曲家 | 体裁/时期 | 听辨特征
  （乐器/速度/情绪/有无人声）。只从课件文字整理，不试听音频。
- **识图题** → 图示要点表（结合"看图"步骤把图里信息文字化）。
- **计算/公式题** → 公式速查表：公式 | 适用条件 | 变量含义。
- **材料/案例分析** → 案例要点表。

## 四、前置汇总文件（全课程共用，放手册最前面）
- `00_总索引.md`：**全课程名词 → 在第几讲**。按"实体类别"分几张表
  （和每讲第四~六节对应）。这是开卷第一入口。维护规则：每整理完一讲就把
  本讲名词并进对应表，**同一名词只留一行**（多讲出现就写多个讲次）。
- `00_思考题汇总.md`（若有大题）：各讲思考题 + 完整参考答案，按讲排列。
- 题型专项总表（若有）：如 `00_必听歌单.md`，把各讲专项素材汇总成一张大表。

> 编译时这些 `00_` 文件会被**按拼音排序**（像字典），各讲正文表保持课程顺序。

## 五、Word 兼容写法（成品要进 Word，别用花哨语法）
只用：`#`/`##`/`###` 标题、`-` 列表、Markdown 表格、`**加粗**`。
不要用：HTML 标签、脚注、嵌套表格、图片外链、`> ` 之外的复杂引用。
表格每格内容尽量短；长内容拆成多行 `-` 列表或多列。
表格必须带 `|---|` 分隔行，否则 build_word 不当它是表格。
**表格单元格里的 LaTeX 公式不能含字面 `|`**（会按列裂表）：绝对值用
`\lvert x \rvert`、范数用 `\lVert x \rVert`，效果相同但无 `|` 字符。

## 六、编译成纸质手册
全部讲整理完后，用脚本合并：
```bash
PYTHONIOENCODING=utf-8 python <skill>/scripts/build_word.py "<课程目录>"
```
脚本会读目录下的 `_manifest.json`（建规则阶段生成，定标题/顺序/哪些表排序），
产出：封面+使用说明、拼音排序的总索引、每讲另起一页、**页眉显示当前讲名**
（STYLEREF 用中文样式名"标题 1"）、表头跨页重复、页脚页码。

`_manifest.json` 示例见 `assets/manifest_example.json`。

### 成品自检
- 总索引四类表是否按拼音排好、无重复行？
- 每讲是否都有"易混淆/不包含"列？
- 思考题答案是否完整到能直接抄？
- 页眉是否正常显示讲名（不是报错"应用 Heading 1"）？若报错见下。

### 已知坑
- 中文版 Word 的 STYLEREF 域必须用本地化名 **"标题 1"**，英文 "Heading 1"
  会每页报错。脚本已用"标题 1"，并给封面设了不同页眉避免首页报错。
- 若 docx 正在 Word 中打开，覆盖保存会失败，脚本自动改存 `_新.docx`。
- 打开成品后按 Ctrl+A → F9 刷新一次页码/页眉域。
